# Projet - CV Simple en HTML/CSS

## Objectif

Créer une page web présentant votre CV (Curriculum Vitae) en utilisant uniquement **HTML et CSS**. Ce projet vous permettra de maîtriser les bases de la structuration HTML et du stylage CSS.

## Description du Projet

Vous devez concevoir une page web statique qui présente votre parcours professionnel et académique de manière claire et attractive. Le CV doit être responsive et s'adapter aux différentes tailles d'écran (desktop, tablette, mobile).

## Contraintes Techniques

### Obligatoire
- **HTML5 sémantique** : Utilisez les balises appropriées (`<header>`, `<section>`, `<article>`, `<footer>`, etc.)
- **CSS3** : Stylisation complète sans framework CSS (pas de Bootstrap, Tailwind, etc.)
- **Responsive Design** : Le CV doit être lisible sur mobile, tablette et desktop
- **Pas de JavaScript** : Ce projet se concentre sur HTML et CSS uniquement

### Technologies Autorisées
- HTML5
- CSS3 (Flexbox et/ou Grid recommandés)
- Google Fonts (optionnel)
- Font Awesome ou équivalent pour les icônes (optionnel)

## Contenu Minimum Attendu

Votre CV doit contenir **au minimum** les sections suivantes :

**1. En-tête (Header)**
   - Votre nom et prénom
   - Titre professionnel ou objectif de carrière
   - Photo professionnelle (optionnel)
   - Coordonnées (email, téléphone, localisation)
   - Liens vers réseaux professionnels (LinkedIn, GitHub, portfolio...)

**2. Résumé/Profil**
   - Courte présentation (2-3 phrases)
   - Vos points forts ou votre proposition de valeur

**3. Expérience Professionnelle**
   - Nom de l'entreprise/organisation
   - Poste occupé
   - Période (dates de début et fin)
   - Description des missions et réalisations
   - (Au minimum 2 expériences, même si fictives pour l'exercice)

**4. Formation**
   - Établissement
   - Diplôme obtenu
   - Année d'obtention
   - (Au minimum 2 formations)

**5. Compétences Techniques**
   - Langages de programmation
   - Technologies et frameworks
   - Outils et logiciels
   - (Représentation visuelle encouragée : barres de progression, badges...)

**6. Compétences Transversales (Soft Skills)**
   - Communication, travail d'équipe, autonomie...

**7. Centres d'Intérêt** (optionnel mais recommandé)
   - Hobbies, activités extra-professionnelles

## Critères de Design

- **Clarté** : L'information doit être facilement lisible
- **Hiérarchie visuelle** : Utilisez différentes tailles de police, couleurs, espacements
- **Cohérence** : Palette de couleurs harmonieuse (2-3 couleurs maximum)
- **Professionnalisme** : Évitez les couleurs trop vives ou les animations excessives
- **Originalité** : Démarquez-vous avec un design personnel (tout en restant professionnel)

## Suggestions et Pistes

### Mise en Page
- **Disposition en colonnes** : Une colonne principale (expériences, formation) et une sidebar (compétences, contacts)
- **Timeline verticale** : Pour l'expérience et la formation
- **Grille de cartes** : Pour les compétences ou projets
- **Header pleine largeur** avec photo de fond (overlay semi-transparent)

### Éléments Visuels
- **Icônes** : Utilisez Font Awesome pour email, téléphone, localisation, GitHub, LinkedIn...
- **Barres de progression** : Pour visualiser le niveau de compétences
- **Badges** : Pour les technologies maîtrisées
- **Bordures et ombres** : Pour faire ressortir les sections
- **Transitions CSS** : Effets au survol (hover) sur les liens

### Responsive Design
- **Mobile First** : Commencez par concevoir pour mobile, puis adaptez pour desktop
- **Media Queries** : Minimum 2 breakpoints (mobile, desktop)
- **Flexbox ou Grid** : Pour des layouts flexibles
- **Typographie adaptative** : Ajustez les tailles de police selon l'écran

### Palette de Couleurs (exemples)
- **Classique** : Noir, blanc, gris avec une couleur d'accent (bleu marine, bordeaux)
- **Moderne** : Fond clair, texte sombre, accent coloré vif
- **Minimaliste** : Niveaux de gris avec une seule couleur d'accent

## Étapes Suggérées

**1. Planification**
   - Dessinez un wireframe (croquis) sur papier
   - Choisissez votre palette de couleurs
   - Listez le contenu à inclure

**2. Structure HTML**
   - Créez la structure sémantique de base
   - Ajoutez tout le contenu (texte, liens...)
   - Validez votre HTML avec le [W3C Validator](https://validator.w3.org/)

**3. Stylisation CSS**
   - Commencez par un reset CSS ou normalize.css
   - Définissez les variables CSS (couleurs, polices...)
   - Stylez section par section
   - Testez régulièrement dans le navigateur

**4. Responsive**
   - Testez sur différentes tailles d'écran
   - Ajoutez les media queries nécessaires
   - Vérifiez la lisibilité sur mobile

**5. Finitions**
   - Peaufinez les espacements
   - Ajoutez des effets de hover
   - Optimisez les images (compression)
   - Testez sur plusieurs navigateurs

## Critères d'Évaluation

- **HTML sémantique** : Utilisation appropriée des balises
- **Qualité du CSS** : Code propre, organisé, commenté
- **Responsive** : Adaptation mobile/desktop
- **Design** : Esthétique, cohérence, professionnalisme
- **Contenu** : Complétude des informations
- **Validité du code** : Code HTML/CSS valide (W3C)

## Compétences Travaillées

- Stylisation CSS3 avancée
- Flexbox et/ou CSS Grid
- Responsive Design et Media Queries
- Typographie et hiérarchie visuelle
- Gestion des couleurs et harmonie
- Optimisation d'images
